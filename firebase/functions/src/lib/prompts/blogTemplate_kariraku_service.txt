# 構成と役割のガイド

## 1. title / excerpt

- `title` は H1 タイトル候補です。  
  例：「【実体験あり】{{service.name}}の評判・口コミ・メリット・注意点」
- `excerpt` は 2〜3文程度のリード文です。  
  - 読者が「このサービス、気になってるけど不安」という気持ちを前提に、
  - 記事でわかること（特徴・料金・メリット/注意点・向き不向き）を短く伝えてください。

## 2. toc（この記事でわかること）

- `toc` には、この記事で得られるポイントを 3〜6 個入れてください。
- それぞれ短いフレーズで、日本語で書きます：
  - 例: 「{{service.name}}の基本スペック」「料金と代表プランの目安」「実際のメリットと注意点」「どんな人に向いているか」など。
- 「目次：」や番号は含めず、シンプルなテキストにしてください。

## 3. sections の構成（H2 見出しと本文）

sections には、次の h2 見出しと本文を **この順番で** 入れてください。

1. 「{{service.name}}はどんなサービス？基本のポイント整理」
   - {{service.name}} の概要・得意なカテゴリ・サービスのポジションを整理します。
   - 箇条書きで特徴3〜5点を書き、それぞれ1文程度の補足を添えてください。
   - 読者が「ざっくりどんなサービスか」をつかめるようにします。

2. 「{{service.name}}を使って良かった点（メリット）」
   - メリットを3〜5個、箇条書きでまとめてください。
   - 各メリットには、短い根拠や具体例（レビュー要約や代表的な使い方）を1〜2文ずつ添えます。
   - 例：初期費用が抑えられる・設置や回収が楽・短期利用しやすい など。

3. 「気をつけたい点・デメリットと、その付き合い方」
   - デメリットや注意点を2〜4個まとめます。
   - 例：長期利用だと割高になる可能性・解約タイミング・破損時のルール・対応エリアの制限 など。
   - それぞれに「どういう人なら問題になりにくいか」「事前に確認しておくと良いポイント」を書き、回避策も軽く添えてください。

4. 「料金と代表プランの目安」
   - 代表的なプランを 2〜4 パターン取り上げ、
     「期間 × 月額 × 総額」がざっくり分かるように Markdown表でまとめてください。
   - 最低利用期間・途中解約・延長・買取の可否など、料金まわりの条件も整理してください。

5. 「申し込み〜設置〜故障時〜回収までの流れ」
   - 1. 申し込み → 2. 配送/設置 → 3. 利用中・故障時のサポート → 4. 回収（延長・買取相談）  
     といった流れを、番号付きリストや小見出しで説明します。
   - 読者が「実際に使うイメージ」を持てるように、生活シーンを少し交えてください。

6. 「{{service.name}}が向いている人 / 向いていない人」
   - 「向いている人」を3点、「向いていないかもしれない人」を3点、箇条書きで整理してください。
   - 向いていない人に対しては、代替として競合サービスを軽く紹介します：
     - 例：  
       `> 競合サービスも含めて検討したい場合は、`  
       `> [{{competitors.[0].name}}の評判](/blog/{{competitors.[0].slug}}) や`  
       `> [{{competitors.[1].name}}の評判](/blog/{{competitors.[1].slug}}) もチェックしてみてください。`

7. 「迷ったときの判断軸と、比較ページへの案内」
   - 迷っている読者に向けて、「予算・期間・ライフスタイル・家事の負担感」などの観点から、
     どう考えると決めやすいかを整理します。
   - 本文の中で、次のような一文を必ず1回入れてください（文言は少しアレンジしてもOK）：
     - 例：  
       「もし“他のサービスとも比べてみたいな…”と思ったら、[家電レンタル3社の比較ページを見る](/blog/{{compareSlug}}) から、全体像を軽くチェックしてみても良いかもしれません。」
   - さらに、本文の最後のほうに **1行だけ** 次のような CTA 行を単独で入れてください：
     - `:::cta[{{service.name}}を詳しく見る（公式サイトへ）]({{service.affiliateUrl}})`

bodyMd は Markdown 形式で書いてください。
H1（#）は使わず、H2（##）とH3（###）を使って読みやすく整理します。

## 4. slugKeys

- 読者が検索しそうなキーワードを 3〜6 個、日本語で入れてください。
- 例：
  - 「{{service.name}} 口コミ 評判」
  - 「{{service.name}} 家電レンタル 料金」
  - 「家電レンタル 初期費用 抑える」 など。

## 5. FAQ

- `faq` には 3〜5 個の質問と回答を入れてください。
- 例：
  - 「Q. 最低利用期間より前に解約するとどうなりますか？」
  - 「Q. 故障した場合の対応はどうなっていますか？」
  - 「Q. 一人暮らしでも利用して良いのでしょうか？」
  - 「Q. 配送や設置にはどのくらい日数がかかりますか？」 など。
- 回答は、事実に基づきつつ、読者を責めずに不安を和らげるトーンで書いてください。

## 6. CTA

- `label`: ボタンに載せるテキスト。例：
  - 「{{service.name}}を公式サイトで詳しく見る」
- `note`: Markdown で 2〜3行の説明を書き、その中に {{service.affiliateUrl}} へのリンクを **1つだけ** 含めてください。
  - 例：

    {{service.name}}の料金プランや対応エリア、キャンペーンの最新情報は公式サイトで確認できます。  
    [{{service.name}}を公式サイトで詳しく見る]({{service.affiliateUrl}})

- `note` の中では、生のURLではなく必ず `[テキスト]({{service.affiliateUrl}})` の形式でリンクを書いてください。

# 禁止事項

- JSON 以外のテキストを一切出力しない
- フィールド名のスペルミスをしない
- schemaにないフィールド（author, date, image, metaTitle など）を追加しない
- 本文やCTA内で `/blog/{{compareSlug}}` や `{{service.affiliateUrl}}` を **単なるテキストとして** そのまま書かない  
  （必ず `[テキスト](URL)` または `:::cta[テキスト](URL)` の形で使う）
- 「絶対にお得」「今すぐ申し込まないと損」など、過度な煽りや誇張表現をしない
