以下の制約にしたがって、JSONだけを出力してください。

# 出力形式（必須・この形だけ）

{
  "title": string,
  "excerpt": string,
  "slugKeys": string[],
  "toc": string[],
  "sections": [
    { "h2": string, "bodyMd": string },
    ...
  ],
  "faq": [
    { "q": string, "a": string },
    ...
  ],
  "cta": {
    "label": string,
    "note": string
  }
}

他のプロパティは絶対に含めないでください。

# 記事の目的

- サイト: {{site.displayName}}（ID: {{site.id}}）
- 想定読者（ペルソナ）: {{persona}}
- いま抱えている悩み: {{pain}}
- トピック: {{topic}}

「年末の料理がしんどい」「共働きで家事が回らない」といった
“家事・料理の負担”に悩む人に向けて、

1. 感情に寄り添って安心させる
2. 具体的な工夫・時短アイデアを提案する
3. 必要に応じて「家電レンタル・サブスク」という選択肢を紹介する
4. 最後に、家電レンタル3社比較ページ（{{compareUrl}}）へ自然に誘導する

という流れの「悩み解決ガイド記事」を作ってください。

# 構成と役割のガイド

## 1. タイトル・リード

- `title` は H1 タイトル候補です。{{topic}} や悩みキーワードを自然に含めつつ、
  「しんどさが少し軽くなりそう」というイメージが伝わるようにしてください。
- `excerpt` は 2〜3文程度のリード文です。  
  読者の今のしんどさにそっと寄り添い、「この記事でできること（整理・工夫・選択肢）」を短く伝えてください。

## 2. スキャンしやすい「この記事でわかること」(toc)

- `toc` には、この記事で得られるポイントを 3〜6 個入れてください。
- それぞれ短いフレーズで、日本語で書きます：
  - 例: 「負担の正体をやさしく整理する」「今日からできる小さな工夫」「家電レンタル・サブスクを“道具のひとつ”として考える」など。
- `toc` の各要素は「目次:」という文字や番号を含めないでください。
- ここは「一気に全体像がつかめる要約リスト」というイメージで作成してください。

## 3. sections の構成（H2 見出しと本文）

sections には、次の h2 見出しと本文を **この順番で** 入れてください。

1. 「{{topic}} に悩むあなたへ。今のしんどさを言葉にしてみましょう」
   - 読者の状況に共感し、「しんどいのはあなただけではない」と安心させる
   - 一日の流れや、心の中のモヤモヤをやさしく描写する
   - **ここで「目次」や箇条書きの章立ては書かず、あくまで共感パートにしてください。**

2. 「まずは負担の正体をやさしく整理してみる」
   - 時間・体力・精神的な負担など、悩みを分解して言語化する
   - 読者が「自分の負担はどこにあるのか」を静かに整理できるように書く

3. 「今日からできる小さな工夫と時短アイデア」
   - 箇条書きを中心に、すぐ試せる具体的なアイデアを書いてください：
     - 例：夕食の作り置きや下味冷凍、買い物頻度の見直し、宅配・ミールキットの活用、
       家族との分担の工夫など
   - 「がんばりすぎない工夫」「完璧じゃなくていい」というメッセージを混ぜてください。

4. 「家電レンタル・サブスクという選択肢」
   - 無理なく頼れる“道具のひとつ”として、家電レンタル・サブスクという選択肢を紹介します。
   - 特定の企業名を連呼せず、「こういうタイプのサービス」というレベルで説明してください。
   - メリット・注意点を中立的に整理してください：
     - メリット例：初期費用を抑えられる・お試し利用ができる・設置や回収を任せられる など
     - 注意点例：長期利用だと割高になる可能性・解約タイミング・破損時のルール など

5. 「迷ったら、比較しながら“自分のペース”で選べばOK」
   - 「必ずレンタルが正解」という書き方は避け、「買う・借りる・現状維持など、選択肢を並べて考える」というスタンスで書いてください。
   - 迷ったときの考え方（予算・期間・ライフスタイル・気持ちの軽さ）を整理します。
   - 文中のどこかで、次のような一文を必ず入れてください（文言は多少アレンジ可）：
     - 例: 「もし“ちょっと気になるかも…”と思ったら、[3社の比較ページを見る]({{compareUrl}}) から、違いを軽くチェックしてみても良いかもしれません。」
   - さらに、本文の最後のほうに **1行だけ** 次のような CTA 行を単独で入れてください：
     - `:::cta[家電レンタル3社の比較ページを見る]({{compareUrl}})`
   - これにより、フロント側でボタンとして表示されます。

bodyMd は Markdown 形式で書いてください。
箇条書きや小見出し（###）を適度に使って読みやすくしてください。

## 4. slugKeys

- 読者が検索しそうなキーワードを 3〜6 個、日本語で入れてください。
- 例: 「年末 料理 しんどい」「共働き 家事 負担 減らす」「家電レンタル 初期費用 抑える」など。
- 1要素あたり、2〜6語くらいのフレーズで構いません。

## 5. FAQ

faq には 2〜4 個の質問と回答を入れてください。

- 例:
  - 「Q. 家事を外注するのは甘えでしょうか？」
  - 「Q. 家電レンタルのデメリットはありますか？」
  - 「Q. 一人暮らしでも家電レンタルは使ってもいいのでしょうか？」 など。
- 回答は、読者を責めず、「こういう考え方もあります」というスタンスで書いてください。

## 6. CTA

cta は次の方針で作ってください。

- `label`: ボタンに載せるテキスト。例：
  - 「家電レンタル3社を比較して、自分に合うサービスを探す」
- `note`: Markdown で 2〜3行の説明を書き、その中に {{compareUrl}} へのリンクを **1つだけ** 含めてください。
  - 例:

    家電レンタル各社の料金やサービス内容をまとめて比較しました。  
    [3社の比較ページを見る]({{compareUrl}})

- `note` の中で `/compare` のような生のパスだけを書くのはやめて、必ず Markdown リンク `[テキスト]({{compareUrl}})` の形式にしてください。

# 禁止事項

- JSON 以外のテキストを一切出力しない
- フィールド名のスペルミスをしない
- schemaにないフィールド（author, date, image など）を追加しない
- 本文やCTA内で `/compare` や `{{compareUrl}}` を **単なるテキストとして** そのまま書かない  
  （必ず `[テキスト]({{compareUrl}})` または `:::cta[テキスト]({{compareUrl}})` の形で使う）
- 「絶対にお得」「今すぐ申し込まないと損」など、過度な煽りや誇張表現をしない
